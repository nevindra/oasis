# Migration: 0.16 → 0.17 (Plan A — Agent foundation)

This release is the first slice of the Agent Foundation Redesign. It changes the public surface for how you call agents and how you compose them. There are no back-compat shims.

## Quick map

| If you wrote… | Change to… |
|---|---|
| `agent.NewLLMAgent(name, desc, p, opts...)` | `agent.New(name, desc, p, opts...)` |
| `network.NewNetwork(name, desc, p, opts...)` | `network.New(name, desc, p, opts...)` |
| `workflow.NewWorkflow(name, desc, opts...)` | `workflow.New(name, desc, opts...)` |
| `a.ExecuteStream(ctx, task, ch)` | `a.Execute(ctx, task, core.WithStream(ch))` |
| `a.ExecuteWith(ctx, task, opts)` | `a.Execute(ctx, task, agent.WithOverrides(opts))` |
| `a.ExecuteStreamWith(ctx, task, ch, opts)` | `a.Execute(ctx, task, core.WithStream(ch), agent.WithOverrides(opts))` |
| `agent.StartStream(ctx, ag, task)` | `agent.Subscribe(ctx, ag, task)` |
| `agent.StartStreamWith(ctx, ag, task, opts)` | `agent.Subscribe(ctx, ag, task, agent.WithOverrides(opts))` |
| Type-asserted on `StreamingAgent` / `AgentWithOptions` / `StreamingAgentWithOptions` | Delete the assertion. Every `Agent` accepts every `RunOption`. |
| Embedded `agent.AgentCore` in your own struct | The runtime spine is internal now. Compose with `network.New` instead of embedding. |
| Imported `agent.Foo` (alias re-export) | Import `core.Foo` (or `processor.Foo`, `skills.Foo`) directly. Code using `oasis.Foo` is unaffected. |
| Used `oasis.StreamingAgent` / `oasis.StartStream` etc. | Use `oasis.Agent` and `oasis.Subscribe` (umbrella re-exports of the new APIs). |

## Why

The old four-way interface ladder and the parallel `ExecuteStream`/`ExecuteWith`/`ExecuteStreamWith` methods forced every Agent implementation to repeat the same four-method skeleton. The new single `Execute(ctx, task, ...RunOption)` form gives the same expressive power with one method to implement, one method to call, and no type-assertion ladders.

## What's NOT changed in this release

This release is the foundation only. The bigger changes — Network as a graph runtime, Workflow as a composable Agent, middleware unification, skills trim, transparent result chunking — are landing in subsequent releases (Plans B / C / D).
