# Migration: 0.17 → 0.18 (Plan B — Network as orchestration primitive)

This release turns `Network` into the framework's single orchestration primitive. The parallel orchestration paths (`agent.WithAgents`, `agent.WithSubAgentSpawning`, the `spawn_agent` builtin tool) are deleted in favor of Network-level features: positional-varargs children, supervisor policies, dynamic membership, and dynamic spawning. There are no back-compat shims.

## Quick map

| If you wrote… | Change to… |
|---|---|
| `network.New(name, desc, p, agent.WithAgents(a, b))` | `network.New(name, desc, p, a, b)` |
| `network.New(name, desc, p, agent.WithAgents(a, b), agent.WithTracer(t))` | `network.NewWithOptions(name, desc, p, []core.Agent{a, b}, network.WithRouter(agent.WithTracer(t)))` |
| `agent.WithSubAgentSpawning(agent.MaxSpawnDepth(2))` on an LLMAgent | `network.WithDynamicSpawning(network.SpawnPolicy{MaxChildren: 10, ChildBuilder: func(req network.SpawnRequest) (core.Agent, error) { ... }})` on a Network |
| `agent.WithAgents(a, b)` on an `agent.LLMAgent` (was already a no-op — LLMAgent ignored it) | Delete the option. If you wanted multi-agent, wrap the LLMAgent in `network.New(...)`. |
| Custom code that detected the `spawn_agent` tool call name | The detection now happens inside Network's dispatch path. You do not need to handle it manually. |
| Custom code that read `oasis.WithSubAgentSpawning`, `oasis.MaxSpawnDepth`, `oasis.DenySpawnTools`, `oasis.WithAgents` | All four re-exports were deleted. Use the migration paths above. |

## Why

Pre-Plan-B, the framework had three parallel paths for "agent A talks to agent B":
- `network.New(..., agent.WithAgents(...))`
- `agent.New(..., agent.WithAgents(...))` (a no-op on LLMAgent — silently dropped)
- `agent.New(..., agent.WithSubAgentSpawning(...))` (LLM-driven spawning via the `spawn_agent` builtin)

All three did roughly the same job under the hood. Plan B unifies them: one Network with first-class orchestration features.

## New surface added

- `network.NewWithOptions(name, desc, router, children, opts...)` — full-featured constructor.
- `network.Option` — the unified option type used by every `With*` below.
- `network.WithRouter(opts ...agent.AgentOption)` — forwards router-side agent options through to the Network's router LLM.
- `network.WithSupervisor(policy)` — applies a `SupervisorPolicy` to every child.
- `network.WithSupervisorFor(name, policy)` — applies a policy to a specific child; repeated calls compose via `Chain`.
- `network.RestartOnFail(max)`, `network.Fallback(backup)`, `network.Quorum(askN, threshold, members...)`, `network.CircuitBreaker(threshold, cooldown)`, `network.Chain(policies...)` — built-in supervisor policies.
- `network.WithParallelDispatch(mode)` — opt out of parallel tool dispatch (`ParallelDisabled`) or use the framework default (`ParallelDefault` / `ParallelByDefault`). The framework already dispatches tool calls in parallel by default; this option is documentation of intent and an opt-out switch.
- `network.AddAgent(child) error` / `network.RemoveAgent(name) error` — thread-safe runtime membership.
- `network.WithDynamicSpawning(policy)` + `network.SpawnPolicy` + `network.SpawnRequest` — LLM-driven sub-agent creation with policy-controlled limits and a user-supplied builder.
- `network.Topology() Topology` — read-only graph snapshot (nodes, edges, applied supervisor summaries).
- `network.ErrCircuitOpen` — error returned when a circuit-breaker-wrapped child is called while the circuit is open.

## What's NOT changed in this release

- `agent.Spawn(ctx, agent, task, opts...) *AgentHandle` — the concurrent-execution wrapper (different from `spawn_agent`) is unchanged. It runs an Agent in a goroutine and returns a handle for cancellation / wait. See `agent/handle.go`.
- LLMAgent's API is unchanged apart from the deletion of `WithAgents` (no-op) and `WithSubAgentSpawning` (replaced by Network's dynamic spawning).
- Workflow's API — Plan C.
- Middleware unification, skills trim, stream consolidation — Plan D.

## Examples

### Two-agent research network — before and after

**Before:**
```go
search := agent.New("search", "Searches", p)
summary := agent.New("summary", "Summarizes", p)
net := network.New("research", "Research team", routerP,
    agent.WithAgents(search, summary))
```

**After:**
```go
search := agent.New("search", "Searches", p)
summary := agent.New("summary", "Summarizes", p)
net := network.New("research", "Research team", routerP, search, summary)
```

### Network with retry policy on every child + router-side tracer

```go
net := network.NewWithOptions("research", "Research team", routerP,
    []core.Agent{search, summary},
    network.WithSupervisor(network.RestartOnFail(3)),
    network.WithRouter(agent.WithTracer(myTracer)),
)
```

### Per-child policy composition

```go
net := network.NewWithOptions("team", "team", routerP,
    []core.Agent{search, flaky},
    network.WithSupervisor(network.RestartOnFail(2)),             // applies to both
    network.WithSupervisorFor("flaky", network.Fallback(backup)), // flaky also gets a fallback
)
```

### Dynamic spawning replaces `WithSubAgentSpawning`

**Before:**
```go
agent.New("router", "...", p,
    agent.WithSubAgentSpawning(agent.MaxSpawnDepth(2)))
```

**After:**
```go
net := network.NewWithOptions("team", "Self-organizing team", routerP, nil,
    network.WithDynamicSpawning(network.SpawnPolicy{
        MaxChildren: 10,
        ChildBuilder: func(req network.SpawnRequest) (core.Agent, error) {
            return agent.New(req.Name, req.Description, p,
                agent.WithPrompt(req.Prompt)), nil
        },
    }),
)
```

The `ChildBuilder` is your hook for whatever construction logic you want — pick the provider, attach tools, wire memory, anything. The framework only enforces `MaxChildren`.

### Inspecting the graph

```go
top := net.Topology()
fmt.Println("Root:", top.Root)
for _, node := range top.Nodes {
    fmt.Printf("  child %s (%s)\n", node.Name, node.Kind)
    for _, sup := range node.Supervisors {
        fmt.Printf("    supervised by %s %v\n", sup.Kind, sup.Params)
    }
}
```

## Divergences from the design spec

- `agent.Spawn` (concurrent-execution wrapper) is **kept** — different feature from `spawn_agent` (LLM-driven creation). See `agent/handle.go`.
- `WithParallelDispatch` is binary (`ParallelDefault` / `ParallelDisabled`). The framework already dispatches in parallel by default; this option is the user-facing knob to disable it.
- `Topology` is read-only and shallow. `LastTrace()` / `Trace.Replay()` are deferred; add in a follow-up if needed.
- `SpawnPolicy.ChildBuilder` runs synchronously on the dispatch path (blocking the router LLM's next iteration until the child is registered).
- `agent/routing.go` was kept (plan said delete) — `buildRoutingSummary` is shared by both LLMAgent and Network via `agent/iteration.go`.
