# Migration: 0.19 → 0.20 (Plan D — Surface Tightening)

This release is the final cleanup pass in the Agent Foundation Redesign series. No new architecture — just deletions, renames, and surface tightening across streams, suspend, skills, workflow aliases, memory types, and the `oasis.go` umbrella.

---

## Quick map

| If you wrote… | Change to… |
|---|---|
| `agent.Suspend(payload json.RawMessage)` | `SuspendProtocol[Req, Resp].Suspend(typedReq)` — typed suspend only |
| `oasis.Suspend(payload)` | Removed. Use `NewSuspendProtocol[Req, Resp]("name").Suspend(req)` |
| `tool.RegisterReadFullResult(reg)` | Delete the call — large results auto-chunk transparently |
| `"read_full_result"` tool in prompts | Remove references — the tool no longer exists |
| `skills.NewFileSkillProvider(dirs...)` | `skills.FromDir(dirs...)` — old name kept as deprecated alias |
| `skills.ChainSkillProviders(p1, p2)` | `skills.Chain(p1, p2)` — old name kept as deprecated alias |
| `skills.NewBuiltinSkillProvider()` | `skills.Builtin()` — old name kept as deprecated alias |
| `workflow.AgentTask` (alias from workflow pkg) | `core.AgentTask` or `oasis.AgentTask` |
| `workflow.AgentResult` (alias from workflow pkg) | `core.AgentResult` or `oasis.AgentResult` |
| `workflow.StringAttr(k, v)` | `core.StringAttr(k, v)` |
| `workflow.IntAttr(k, v)` | `core.IntAttr(k, v)` |
| `workflow.Float64Attr(k, v)` | `core.Float64Attr(k, v)` |
| `memory.ItemStore` | `core.MemoryItemStore` — `memory.ItemStore` kept as deprecated alias |
| `oasis.X` for any dropped umbrella name | Import the subpackage directly (see "Trimmed oasis.go" below) |

## What's NOT changed

- `core.Agent`, `LLMAgent`, `Network`, `Workflow` — all semantics identical.
- `*ErrSuspended` — still returned; `errors.As` consumers are unaffected.
- `agent.WithRetry` / `ratelimit.WithRateLimit` — still work; now also available as composable middleware via `agent.RetryMiddleware` / `ratelimit.RateLimitMiddleware`.
- `ToolResultStore` / `WithToolResultStore` — still available for post-hoc inspection.
- All `WorkflowContext` public methods (`Get`, `Set`, `Input`, `Resolve`, `ResolveJSON`).
- `WorkflowResult.Status`, `Steps`, `Usage`.
- `oasis.NewWorkflow`, `oasis.NewAgent`, `oasis.NewNetwork` — unchanged.

## Why

### Suspend unified

`agent.Suspend(json.RawMessage)` was an untyped escape hatch that predated `SuspendProtocol[Req, Resp]`. Every real caller was already using the typed form. Keeping the untyped form was a liability: callers had to do their own marshaling/unmarshaling, schema was invisible to introspection, and two suspend paths meant two code paths to test. The typed protocol is the single path now.

### Transparent chunking (read_full_result removed)

`read_full_result` was a workaround: when a tool returned more tokens than the context window safely allowed, users registered this special tool so the model could page through results. The new approach is simpler — large tool results are split into consecutive tool-result messages automatically, using rune-safe UTF-8 splitting. No tool registration, no model prompt changes, no user opt-in.

### Skills trim

`NewFileSkillProvider`, `ChainSkillProviders`, and `NewBuiltinSkillProvider` used a Java-style "New prefix + verbose noun phrase" pattern inconsistent with the rest of the framework. The new names — `FromDir`, `Chain`, `Builtin` — match Go conventions and are shorter to type. Old names are kept as deprecated wrappers for a smooth upgrade path.

### Workflow aliases removed

`workflow/workflow.go` had 12 local type aliases (`type AgentTask = core.AgentTask`, etc.) and 3 attribute helpers that were pure copies of `core` exports. They created two paths to reach the same type, which caused confusion in imports and made `go doc` output noisy. Workflow internals now import `core` directly; callers should use `oasis.AgentTask` or `core.AgentTask`.

### MemoryItemStore moved to core

`memory.ItemStore` was defined in the `memory` package but implemented by `store/sqlite` and `store/postgres` — satellite modules that depend on `core`, not `memory`. Moving the interface to `core` breaks the import cycle and lets store satellites declare compliance without pulling in the full memory package. `memory.ItemStore` remains as a deprecated alias so existing code compiles unchanged.

### oasis.go curated down

The umbrella grew organically to 838 lines / 246 declarations. Most of those were "copy everything from the subpackage" re-exports that made `oasis.go` a maintenance burden and obscured what the framework actually recommends for day-one usage. The new rule: `oasis` re-exports only what a developer needs in the first 30 minutes. Power-user APIs stay in their subpackage. See "Trimmed oasis.go" for the complete list.

### Middleware unified (provider + agent + tool)

Three middleware layers now share a consistent shape:

- `provider.Middleware = func(core.Provider) core.Provider` + `provider.Chain(...)`
- `agent.Middleware = func(core.Agent) core.Agent` + `agent.Chain(...)`
- `core.ToolMiddleware` + `core.ChainToolMiddleware(...)`

Each layer is wired in by one option: `agent.WithMiddleware(...)`.

## Examples

### Suspend — before and after

**Before (untyped — removed):**
```go
err := agent.Suspend(json.RawMessage(`{"question":"Approve?"}`))
```

**After (typed protocol):**
```go
// Declare once (e.g., package-level).
var approvalProtocol = oasis.NewSuspendProtocol[ApproveReq, ApproveResp]("approval")

// Inside a tool or step:
err := approvalProtocol.Suspend(ApproveReq{Question: "Approve?"})

// Resume handler (unchanged from 0.19):
var req ApproveReq
resp, err := approvalProtocol.Resume(ctx, suspendErr, ApproveResp{Approved: true})
```

`errors.As` on `*oasis.ErrSuspended` still works without changes — only the `Suspend()` call site changes.

---

### Skills — before and after

**Before:**
```go
provider := skills.NewFileSkillProvider("./skills")
chain := skills.ChainSkillProviders(provider, skills.NewBuiltinSkillProvider())
ag := agent.New("bot", "...", p, agent.WithSkills(chain))
```

**After:**
```go
provider := skills.FromDir("./skills")
chain := skills.Chain(provider, skills.Builtin())
ag := agent.New("bot", "...", p, agent.WithSkills(chain))
```

The old names compile with a deprecation notice — you can migrate at your own pace.

---

### Middleware composition — before and after

**Before (options stacked separately):**
```go
p := myProvider
p = ratelimit.WithRateLimit(p, ratelimit.RPM(60))

ag := agent.New("bot", "...", p,
    agent.WithRetry(agent.RetryMaxAttempts(3)),
)
```

**After (composable middleware chain):**
```go
providerMW := provider.Chain(
    ratelimit.RateLimitMiddleware(ratelimit.RPM(60)),
)
agentMW := agent.Chain(
    agent.RetryMiddleware(agent.RetryMaxAttempts(3)),
)

p := providerMW(myProvider)
ag := agent.New("bot", "...", p,
    agent.WithMiddleware(agentMW),
)
```

`agent.WithRetry` and `ratelimit.WithRateLimit` still work and are not deprecated — use whichever form fits your codebase.

---

### Workflow attribute helpers — before and after

**Before:**
```go
import "github.com/nevindra/oasis/workflow"

attrs := []core.Attribute{
    workflow.StringAttr("model", "gpt-4o"),
    workflow.IntAttr("max_tokens", 1024),
}
```

**After:**
```go
import "github.com/nevindra/oasis/core"

attrs := []core.Attribute{
    core.StringAttr("model", "gpt-4o"),
    core.IntAttr("max_tokens", 1024),
}
```

## Trimmed oasis.go

The umbrella dropped from **246 declarations** to **57 declarations**. If you get a compile error like `oasis.X undefined`, import the subpackage where `X` now lives:

| Dropped from `oasis` | Lives in |
|---|---|
| Hook types (`BeforeRunHook`, `AfterRunHook`, etc.) | `github.com/nevindra/oasis/agent` |
| Finish reasons (`FinishReasonStop`, `FinishReasonToolCall`, etc.) | `github.com/nevindra/oasis/core` |
| Event constants and event types (`EventTypeText`, `TextEvent`, etc.) | `github.com/nevindra/oasis/core` |
| Tool middleware constructors (`ApplyToolMiddleware`, `ChainToolMiddleware`) | `github.com/nevindra/oasis/core` |
| Workflow step types (`Step`, `AgentStep`, `ForEach`, `DoWhile`, `DoUntil`) | `github.com/nevindra/oasis/workflow` |
| Memory types (`MemoryConfig`, `MemoryItem`, `MemoryFilter`) | `github.com/nevindra/oasis/memory` or `github.com/nevindra/oasis/core` |
| Persistence/capability interfaces (`Persister`, `ConversationStore`, etc.) | `github.com/nevindra/oasis/core` |
| Provider/catalog types (`ModelInfo`, `CatalogProvider`, etc.) | `github.com/nevindra/oasis/provider/catalog` |
| Tracer/span helpers (`Tracer`, `Span`, `NoopTracer`) | `github.com/nevindra/oasis/core` |
| `Suspend` (untyped) | Removed entirely — use `NewSuspendProtocol[Req, Resp]` |
| `read_full_result` tool | Removed entirely — chunking is automatic |

**Agent options (23):** `WithTools`, `WithPrompt`, `WithMemory`, `WithLimits`, `WithGeneration`, `WithResponseSchema`, `WithDynamicPrompt`, `WithDynamicModel`, `WithDynamicTools`, `WithTracer`, `WithLogger`, `WithMetadata`, `WithPreProcessors`, `WithPostProcessors`, `WithToolPolicy`, `WithToolApproval`, `WithInputHandler`, `WithMiddleware`, `WithSkills`, `WithActiveSkills`, `WithEmbedding`, `WithRetry`, `WithOverrides`

For the authoritative list, read `oasis.go` directly — it is short enough to scan in two minutes.

## Divergences from the design spec

- `WorkflowContext` stays exported (required by `StepFunc` signature). A future `StepFunc` redesign may tighten this.
- `agent.WithRetry` and `ratelimit.WithRateLimit` are not deprecated — they're convenience wrappers over the new middleware constructors and remain first-class.
- Stream file consolidation (`agent/stream*.go` → `agent/stream.go`) is an internal change; public exports (`ServeSSE`, `WriteSSEEvent`, `Stream`, `Subscribe`) are unchanged.
